# 백테스트 검증 체크리스트

백테스트가 **오류 없이**, **선행 참조(look-ahead) 없이** 동작하는지 점검한 내용입니다.

---

## 1. 선행 참조(Look-Ahead) 여부

| 항목 | 검증 결과 |
|------|-----------|
| **장악형 플래그** | `add_engulfing_flags`는 `(i-1, i)` 봉만 사용. i번째 행의 플래그는 i번째 봉 마감 시점에 알 수 있는 정보만 사용. ✅ |
| **전략 신호** | `run_signal_on_bar(i, row, prev_row, df.iloc[:i+1], ...)` — `df`는 **i번째 봉 포함**까지만 전달. i+1 이후 봉 미사용. ✅ |
| **4h 추세** | `merge_asof(..., direction='backward')` — 각 1h 봉에 **그 이전에 이미 마감된** 4h 봉의 종가·EMA50만 매핑. 현재 형성 중인 4h 봉 미사용. ✅ |
| **손절/익절 판단** | 해당 봉의 `high`, `low`만 사용 (그 봉이 마감된 뒤 알 수 있는 정보). ✅ |

**결론: 미래 데이터를 참조하는 부분 없음.**

---

## 2. 같은 봉 내 처리 순서

- **손절 vs 익절**: 같은 봉에서 둘 다 조건 충족 시 **손절을 먼저** 적용 (`strategy.py`에서 stop 체크 후 TP 체크).
- 실거래에서는 봉 내 순서를 모르므로, 보수적으로 “손절이 먼저 터진다”고 가정한 것. ✅

---

## 3. 포지션·손익 계산

| 항목 | 검증 결과 |
|------|-----------|
| **진입 시** | 수수료를 `capital`에서 차감 후, `positions`에 `entry_capital`(수수료 반영 후 자본) 저장. ✅ |
| **노션얼** | `entry_capital * size_pct * leverage` — 진입 시점 자본 기준. ✅ |
| **TP_FIRST** | `tp1_done=False`인 **첫 번째** 포지션의 50%만 청산. 해당 포지션의 `entry`, `entry_capital`, `size_pct` 사용. ✅ |
| **STOP** | 보유 중인 **모든** 포지션을 손절가로 청산, 수수료 적용. ✅ |
| **미실현 손익** | 매 봉 종가 기준으로 보유 포지션 평가. ✅ |

---

## 4. 알려진 제한사항(버그 아님)

1. **첫 봉 누락**  
   `add_engulfing_flags`에서 `df.iloc[1:]`로 첫 행을 잘라내므로, 백테스트에 사용되는 구간은 **원본 데이터의 2번째 봉부터** 시작.  
   → 첫 1봉에서 나올 수 있는 신호는 없음. 영향은 1봉 분량으로 제한적.

2. **봉 내 체결가**  
   1h 봉만 있으므로 봉 **내부**의 체결 시점/가격은 모름.  
   → `CONSERVATIVE_FILL` + `FILL_USE_MID`로 “해당 봉 종가” 또는 “(고가+저가)/2”로만 체결가 산정.  
   → 봉 내 실제 체결가와 차이는 있을 수 있으나, 의도적으로 보수적/중립적으로 설정한 부분.

3. **동일 봉 다중 이벤트**  
   한 봉에서 손절·익절·진입이 여러 번 나올 수 있어도, **한 봉당 하나의 액션**만 처리(먼저 체크된 것만 실행).  
   → 봉 단위 백테스트의 일반적인 가정과 동일.

---

## 5. Early Return 반환값

- 데이터 부족 시 `run_backtest`가 `(trades, equity, capital, info)` 4개를 반환하도록 수정됨.
- `info`는 정상 시 `n_bars`, `first_ts`, `last_ts` 포함; 데이터 부족 시 `{}`. ✅

---

## 요약

- **선행 참조 없음**: 모든 판단은 “그 시점까지 마감된 봉”만 사용.
- **손익/포지션 로직**: 진입·청산·수수료·노션얼 계산 일관됨.
- **체결가**: 봉 단위 데이터 한계를 인정하고, 보수적 가정(종가/중간가)으로만 사용.

위 범위 내에서 **백테스트는 의도대로 동작하며, 일반적으로 말하는 “백테스팅 오류”는 범하지 않은 것으로 판단**할 수 있습니다.
